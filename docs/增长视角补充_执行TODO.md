## 1. 目标说明

本轮不是新开一套独立“增长系统”，而是在现有 `Web-Kanban` 五个看板基础上，补最小可落地的增长分析能力。

执行顺序固定为：

1. 先在趋势页补增长链路
2. 再在门店页补少量机会榜
3. 最后再考虑大盘效率总览
4. 地域机会榜后置

当前原则：

- 只使用现有标准化字段
- 不引入当前数据源不支持的新结论
- 不修改既有指标的字段含义
- 每一步都优先做“小闭环”

## 2. 当前可用字段边界

当前可以稳定复用的核心字段：

- `revenue`
- `valid_orders`
- `exposure_users`
- `visit_users`
- `visit_conversion_rate`
- `order_conversion_rate`
- `customer_paid`
- `gross_amount`
- `platform`
- `province`
- `city`
- `standard_store_name`

当前明确不做：

- 新客 / 老客
- 复购分析
- 投放成本 / ROI
- 活动归因
- 利润增长

原因：

- 当前数据层没有对应稳定字段
