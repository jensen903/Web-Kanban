## 2. 客观发现

### 2.1 文件格式不统一

- 美团汇总为 `csv`
- 饿了么汇总为 `xlsx`
- 京东汇总为 `xlsx`
- 美团 `csv` 需按 `gb18030` 编码读取，不能直接按 `utf-8` 读取

### 2.2 日期字段格式不统一

- 美团 `日期` 为 `yyyymmdd` 形式，例如 `20260110`
- 饿了么 `日期` 为标准日期字符串，例如 `2026-01-01`
- 京东 `日期` 为标准日期字符串，例如 `2026-01-01`

### 2.3 平台字段名不统一

- 美团核心收入字段是 `营业收入`
- 饿了么核心收入字段是 `收入`
- 京东核心收入字段是 `收入`
- 美团门店主键是 `门店id`
- 饿了么门店主键是 `门店编号`
- 京东门店主键是 `门店id`
- 地域字段也不一致：
  - 美团：`省份`、`门店所在城市`、`区县市`
  - 饿了么：`省份`、`城市名称`、`区县名称`
  - 京东：只有 `城市`

### 2.4 重新导入后字段缺失问题已部分消除

- 你重新导入后：
  - `饿了么3月汇总.xlsx` 已补齐此前缺失的 `省份`、`营业额`、`支出`、`活动补贴`、`平台技术服务费`、`履约技术服务费` 等主要字段
  - `4月京东汇总.xlsx` 已补齐此前缺失的 `曝光次数`、`入店次数`、`活动订单量`、`活动力度`、`投入产出比`、`日均营业时长`、`履约准时率` 等字段
- 美团 `1-4 月汇总` 的字段结构基本一致

## 5. 对看板开发的直接影响

### 5.1 可以直接支撑的指标

- 总营收
- 总订单量
- 平台营收占比
- 平台订单量对比
- 客单价
- 曝光人数趋势
- 入店/进店转化率
- 下单转化率
- 城市维度排行
- 门店维度排行

### 5.2 当前仍需继续确认的点

- 业务上 `淘宝闪购` 与 `饿了么` 被视为同一平台
  - 展示层统一写为 `饿了么`
  - 底层数据源当前实际读取的是 `饿了么汇总`
- 需求里的 `到手率 = 收入 / 营业额`
  - 饿了么 `1/2/4 月` 有 `营业额`
  - 京东 `1-4 月` 有 `营业额`
  - 美团没有明确名为 `营业额` 的字段，只有 `优惠前总额`、`商品原价`、`顾客实付` 等相关字段
  - 已确认执行口径：`到手率 = 收入 / 营业额`
  - 美团在标准化时需先明确映射出可作为 `营业额` 的字段
- `门店转化率 Top20` 目前不够精确
  - 原始数据中存在 `入店转化率` 和 `下单转化率`
  - 但没有统一名为 `门店转化率` 的字段
  - 已确认执行口径：取 `下单转化率`
- `覆盖省份`
  - 美团和饿了么有省份字段
  - 京东源文件只有城市字段
  - 可通过 `门店对齐表_最新版.xlsx` 中的 `省份/城市/行政区` 信息补齐京东省份

## 6. 风险结论

- 这批数据足够支持 `5 个看板的 Mac 本地演示版`
- 其中以下 2 个点已经明确：
  - `淘宝闪购` 与 `饿了么` 视为同一平台，但展示与数据层统一使用 `饿了么`
  - 京东省份可基于门店对齐表补齐
- 此外以下 2 个口径也已确认：
  - `到手率 = 收入 / 营业额`
  - `门店转化率 Top20` 取 `下单转化率`

- 以当前基线看，`1-8` 阶段的关键业务口径已收口，可以进入后续开发阶段。
