## 2. 分层定位

### 91_经营汇总表

定位：跨平台经营原始汇总层

用途：

- 保留平台原始经营汇总字段
- 作为 92 的上游输入
- 方便回溯平台原始字段和后续补充口径

特点：

- 字段多，保留原始平台差异
- 不建议直接作为第一版业务增长看板数据源

### 92_经营明细表

定位：统一门店后的经营明细层

用途：

- 作为 DataEase 明细分析基础表
- 支持按平台、门店、城市、日期做明细筛选
- 作为 93 的直接输入层

特点：

- 已通过门店主表完成跨平台门店对齐
- 保留日级事实记录
- 是当前推荐的经营明细分析数据源

### 93_增长汇总表

定位：增长看板日级聚合层

用途：

- 支持近 7 天
- 支持自定义时间
- 支持日趋势
- 支持平台 / 城市 / 门店增长贡献分析

特点：

- 以日为基础粒度
- 保留统计月份，方便月度汇总
- 增长指标和贡献指标已经提前计算

## 3. 91 表字段口径

### 核心理解

`91_经营汇总表` 是跨平台原始汇总宽表，来源于京东 / 美团 / 饿了么经营数据文件合并。

它的字段大体分为四类：

- 来源追踪字段
- 时间字段
- 原始门店字段
- 平台经营指标字段

### 关键字段

- `来源平台`
  说明：数据来自哪个平台，取值如 `京东`、`美团`、`饿了么`

- `来源月份`
  说明：来源文件名识别出的月份数字，如 `4`

- `日期_标准`
  说明：标准化后的日级日期，格式如 `2026-04-26`

- `日期月份`
  说明：标准化后的月份标签，格式如 `2026-04`

- `门店名称`
  说明：平台原始门店名称

- `门店id` / `门店编号`
  说明：平台原始门店 ID 字段
  备注：京东、美团主要使用 `门店id`，饿了么主要使用 `门店编号`

- `收入`
  说明：平台原始收入口径字段

- `营业收入`
  说明：部分平台的另一种原始收入口径字段

- `有效订单`
  说明：平台原始有效订单数

- `实付单均价` / `单均实付`
  说明：平台原始客单价字段

### 91 表使用建议

- 作为原始承接层保留
- 不建议直接作为第一版增长看板口径表
- 后续如果 92 要补字段，优先回到 91 取原始列

## 4. 92 表字段口径

### 表定位

`92_经营明细表` 是基于 `91_经营汇总表` 加工出的统一门店明细层。

### 主键理解

一条记录可以理解为：

`平台 + 日期 + 统一门店`

### 字段说明

- `来源平台`
  说明：原始数据平台

- `来源月份`
  说明：来源文件月份数字

- `日期月份`
  说明：月份标签，如 `2026-04`

- `日期_标准`
  说明：日级日期，如 `2026-04-26`

- `日期开始`
  说明：起始日期

- `日期结束`
  说明：结束日期

- `统一门店ID`
  说明：通过门店主表生成的统一门店唯一标识

- `统一门店名称`
  说明：统一后的标准门店名

- `统一省份`
  说明：门店统一省份

- `统一城市`
  说明：门店统一城市

- `统一行政区`
  说明：门店统一行政区

- `统一地址`
  说明：门店统一地址

- `平台门店ID`
  说明：平台原始门店 ID

- `原始门店名称`
  说明：平台原始门店名

- `门店对齐状态`
  说明：`已对齐` 或 `未对齐`

- `收入_统一`
  来源：`收入` 或 `营业收入`
  规则：按字段优先级取非空值
  说明：统一后的收入字段

- `实付_统一`
  来源：`顾客实付` 或 `顾客实付总额`
  规则：按字段优先级取非空值

- `有效订单`
  说明：统一后的有效订单数

- `客单价_统一`
  来源：`实付单均价` 或 `单均实付`
  规则：按字段优先级取非空值

- `活动补贴_统一`
  来源：`活动补贴` 或 `平台活动补贴`
  规则：按字段优先级取非空值

- `支出_统一`
  来源：`支出`
  说明：统一后的支出字段

### 92 表使用建议

- 明细表、门店日明细、城市日明细优先使用 92
- 自定义时间分析优先使用 92
- 93 的所有聚合都从 92 出发

## 5. 93 表字段口径

### 表定位

`93_增长汇总表` 是以日为基础粒度的增长看板聚合层。

### 主键理解

一条记录可以理解为：

`统计日 + 平台 + 统计口径 + 统计对象 + 是否只看已对齐门店`

### 字段说明

- `统计日`
  来源：`92.日期_标准`
  说明：日级日期，如 `2026-04-26`
  用途：近 7 天、自定义时间、日趋势

- `统计月份`
  来源：`92.日期月份`
  说明：月级标签，如 `2026-04`
  用途：月度汇总、月筛选

- `平台`
  说明：当前聚合记录所属平台范围
  取值示例：
  `全部`、`京东`、`美团`、`饿了么`

- `统计口径`
  说明：当前这行是在按什么维度统计
  当前取值：
  `全国`、`平台`、`城市`、`门店`

- `统计对象`
  说明：当前具体统计谁
  示例：
  `全国`、`美团`、`上海市`、某个门店名

- `门店名称`
  说明：当统计口径为门店时填充，否则为空

- `城市`
  说明：当统计口径为城市或门店时填充，否则为空

- `是否只看已对齐门店`
  说明：区分是否只统计 `门店对齐状态=已对齐` 的数据
  取值：
  `是`、`否`

- `收入`
  来源：`92.收入_统一`
  公式：`sum(收入_统一)`
  说明：当前粒度下的收入汇总

- `有效订单数`
  来源：`92.有效订单`
  公式：`sum(有效订单)`
  说明：当前粒度下的有效订单汇总

- `实付客单价`
  来源：`92.客单价_统一`、`92.有效订单`
  公式：`sum(客单价_统一 × 有效订单) / sum(有效订单)`
  说明：聚合后客单价
  备注：这里使用加权算法，不是简单平均

- `上期收入`
  说明：同一平台、同一统计口径、同一统计对象、同一对齐口径下的前一日收入

- `上期有效订单数`
  说明：同条件下前一日有效订单数

- `上期实付客单价`
  说明：同条件下前一日实付客单价

- `收入增长额`
  公式：`收入 - 上期收入`

- `收入增长率`
  公式：`(收入 - 上期收入) / 上期收入`
  备注：上期为 0 或空时置空

- `订单增长额`
  公式：`有效订单数 - 上期有效订单数`

- `订单增长率`
  公式：`(有效订单数 - 上期有效订单数) / 上期有效订单数`
  备注：上期为 0 或空时置空

- `客单价增长额`
  公式：`实付客单价 - 上期实付客单价`

- `客单价增长率`
  公式：`(实付客单价 - 上期实付客单价) / 上期实付客单价`
  备注：上期为 0 或空时置空

- `收入增长贡献额`
  说明：当前对象带来的收入增量，等于 `收入增长额`

- `收入增长贡献占比`
  公式：`当前对象收入增长额 / 同统计日 + 同统计口径 + 同平台 + 同对齐口径下所有对象收入增长额之和`

- `订单增长贡献额`
  说明：当前对象带来的订单增量，等于 `订单增长额`

- `订单增长贡献占比`
  公式：`当前对象订单增长额 / 同统计日 + 同统计口径 + 同平台 + 同对齐口径下所有对象订单增长额之和`
